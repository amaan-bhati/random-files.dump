What is an API?

API = Application Programming Interface

Simple Definition:

An API is like a waiter in a restaurant

  • You (the app) tell the waiter (API) what you want
  • The waiter goes to the kitchen (server/database)
  • The waiter brings back exactly what you ordered

Technical Definition:

A set of rules and protocols that allows different software applications to communicate with each other

Real-World API Analogy

Think of APIs like Electrical Outlets:

üîå

Standardized Interface

  • Every device knows how to plug in
  • Don't need to understand the power grid
  • Just plug in and get electricity

APIs Work Similarly

  • Standardized way to request data
  • Don't need to understand backend complexity
  • Just make a request and get a response

Example: When you use Google Maps in Uber, Uber uses Google Maps API to get location data

How APIs Work - The Communication Flow

1. Request ‚Üí 2. Processing ‚Üí 3. Response

Step 1: The Request

Your app sends a request to an API endpoint with URL, method, headers, and sometimes data

Step 2: Processing

API receives, validates, processes the request and prepares the response

Step 3: The Response

API sends back structured data (usually JSON) with status codes

API Request-Response Example

Scenario: Getting weather information

Request:
GET https://api.weather.com/current?city=Mumbai
Headers: Authorization: Bearer your-api-key
Response:
{
  "city": "Mumbai",
  "temperature": 28,
  "condition": "Sunny",
  "humidity": 65,
  "status": "success"
}

What Happened: Your app asked for Mumbai weather, API processed it, and returned current weather data in JSON format

REST APIs - The Most Popular

REST = Representational State Transfer

Key Principles:

  • Stateless: Each request is independent
  • Resource-based: Everything has a URL
  • HTTP Methods: Standard verbs
  • JSON: Lightweight data format

Why REST is Popular:

  • Simple to understand
  • Works great with web
  • Scalable and cacheable
  • Platform independent

Examples: Twitter API, GitHub API, Instagram API

GraphQL = Query Language for APIs

Key Features:

  • Single Endpoint: One URL for all operations
  • Flexible Queries: Request exactly what you need
  • Strongly Typed: Clear data structure definitions
  • Real-time: Built-in subscriptions
Example Query:
{
  user(id: "123") {
    name
    email
    posts {
      title
      likes
    }
  }
}

Used By: Facebook, GitHub, Shopify, Airbnb

HTTP Methods - The API Verbs

CRUD Operations with HTTP Methods:

GET - Read/Retrieve data

GET /users ‚Üí Get all users | GET /users/123 ‚Üí Get specific user

POST - Create new resources

POST /users ‚Üí Create new user

PUT - Update/Replace entire resource

PUT /users/123 ‚Üí Replace user completely

DELETE - Remove resources

DELETE /users/123 ‚Üí Delete specific user

HTTP Status Codes - API Communication

Status Codes Tell the Story:

2xx - Success

  • 200 OK ‚Üí Request successful
  • 201 Created ‚Üí New resource created
  • 204 No Content ‚Üí Success, no data

4xx - Client Errors

  • 400 Bad Request ‚Üí Invalid format
  • 401 Unauthorized ‚Üí Auth required
  • 404 Not Found ‚Üí Resource missing

5xx - Server Errors

  • 500 Internal Server Error ‚Üí Server broke
  • 503 Service Unavailable ‚Üí Server down

Remember:

Status codes help your app handle different scenarios appropriately

API Security Basics

Keeping APIs Safe:

Authentication Methods:

  • API Keys: Simple tokens
  • OAuth: Secure authorization
  • JWT Tokens: Stateless auth

Security Measures:

  • HTTPS Only: Encrypt communication
  • Rate Limiting: Prevent abuse
  • Input Validation: Check data
  • Access Control: Limit permissions

Example: GitHub requires authentication for private repositories but allows public data without keys
